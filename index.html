PWA proof-of-concept for LTP Questionnaires

Goals

  1. implement manifest so app can be installed to 'home screen'
  2. use a Service Worker to cache content for offline usage
  3. experiment with (mobile) interaction within a questionnaire
  4. use Preact to serve a full experience with a very small page load

Prologue

  • Consider this web application to be NeOn's inbox page where questionnaires can be started. Consider only one questionnaire to be available. All other functionality (ie the top menu) is non-functional.
  • When this application is launched 2 times within 5 minutes on a mobile (Android) device, a 'A2HS' banner appears prompting the user to install the web page as an app on their 'home screen'. This effect can be mimicked on a Desktop computer using Chrome's (v62+) web inspector.
  • Note that the app, when launched from the home screen, is currently identical to the web page, but should behave more like an app instead of a web page (ie no browser bar, full screen mode, orientation lock)
  • When the application detects network loss, it will serve pre-cached versions of the pages. You may notice that some functionality becomes inactive (ghosted) when offline. The goal was to be able to complete a questionnaire while being offline.

In fact, to experience all features, you should go offline now: enable flight mode on your mobile device or stop the docker container on your computer.

Then go to the inbox page using the top navigation. You'll see an overview of your assessments. That would be the start page of the PWA (all data is already loaded there).
